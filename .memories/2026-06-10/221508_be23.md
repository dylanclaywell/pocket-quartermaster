## Art pipeline: server cache as source of truth + ES-DE harvest

Pivoted from "build a scraper" to "harvest art launchers already have, cache server-side, share/push everywhere." Avoids ScreenScraper credentials, rate limits, and ROM hashing (all bad fits for Pi Zero 2 W).

### Key realization
The server art cache **already existed** as the activity thumbnail system (`server/utils/thumbnails.ts`): per-game images keyed by gameKey, served at `/api/thumbnails/<gameKey>`, with download/search/delete endpoints and user-set art already shared across activity + game-detail. So the proposal = two glue pieces + a paradigm tweak, not a big build.

### Built this session
1. **ES-DE root paradigm.** Renamed the uncommitted `esDeGamelistsRelPath` → `esDeRootRelPath` (points at the `ES-DE/` data dir). Derive everything under it: gamelists at `gamelists/<system>/`, art at `downloaded_media/<system>/<arttype>/`. New `esDeSubdir(dest, ...sub)` helper; `metadataTargetRoot` now derives `<root>/gamelists`. Renamed across types.ts, romTransfer SourceRoot, both PATCH endpoints, device page UI (picker now "ES-DE folder", instructions updated).
2. **Harvest** (`server/utils/launcherArtHarvest.ts` + `server/api/roms/art/harvest.post.ts`). Scans `<ES-DE>/downloaded_media/<system>/{covers,miximages,screenshots}/`, matches `(system, filename-no-ext)` → gameKey against catalog variants, `saveThumbnail()` for games with NO cached art (covers/first-art-type wins; user art never clobbered → no provenance store needed). Pi-safe: small-image copies, no network, no hashing. Returns {scanned, imported, skippedExisting, unmatched, reason}.
3. UI: "Import art from device" button on transfer page next to "Sync names", with result note.

### Verification
- `npm run typecheck` clean.
- Dev server: harvest 400 (no body), returns reason "Art import is only supported for ES-DE destinations" for non-es-de dest; transfer + devices pages 200. Server killed, port clear.

### Design decisions locked
- Art granularity = per-game (matches existing cache). Per-variant skipped.
- Precedence: harvest fills empty slots only; user-set/manual always win.
- External scraping (TheGamesDB/ScreenScraper) deferred as optional gap-filler writing into the same cache; needs CRC (still unpopulated in scan) ideally from the future desktop indexer.
