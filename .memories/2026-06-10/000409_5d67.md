### Design decisions reached (saved as active plan `rom-management-feature-design`)
- **Game vs RomFile split.** Game = logical (display name, save profile link); RomFile = physical (filename, CRC32, region tags). No-Intro mental model.
- **Concrete schema for v1**, not the full abstract resolver. Per-device `romsRootRelPath` parallel to existing `retroarchActivityDir`. Logical-layer abstraction kept open as upgrade path.
- **Filename ≠ display name.** Master keeps canonical No-Intro names; launcher metadata writers (ES-DE first, muOS second) handle display strings.
- **Per-(game, device) preferred variant.** Handles "USA on Deck, EUR on RG35XX" without ceremony.
- **Saves key on (game, variant)**, not (game). Prevents corruption across regions.
- **Pi Zero 2 W constraint** saved as separate project memory — no on-device AI, lazy CRC caching, heavy work belongs in a future desktop indexer.
