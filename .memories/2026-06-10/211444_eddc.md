## ES-DE writer: Android pivot — configurable gamelists folder

Reworked the ES-DE name writer after discovering Android ES-DE can't put gamelists in the SD rom folders.

### Discovery
- `LegacyGamelistFileLocation` does NOT relocate gamelists into rom folders on Android (scoped storage). User confirmed: gamelist only appeared on internal storage.
- Android fix: relocate ES-DE's whole **application data directory** to the SD (re-trigger first-run config by revoking storage permission / renaming the ES-DE folder; no in-app move button). Then gamelists live at `<SD>/ES-DE/gamelists/<system>/gamelist.xml` — on the SD, reachable by this app. User got this working.

### Code change (gamelists root is separate from romsRoot now)
- New optional `esDeGamelistsRelPath` on `DeviceIdentity` + `VirtualMount` (forward-slash, relative to mount root, e.g. `ES-DE/gamelists`).
- `SourceRoot` (romTransfer) now exported, carries `mountPath` + `esDeGamelistsRelPath`.
- New `metadataTargetRoot(kind, dest)` in launcherMetadata.ts: ES-DE → `mountPath/esDeGamelistsRelPath`; returns `{reason}` when not mounted or path unset. muOS → mountPath (future).
- `execute.post.ts` + `sync.post.ts` route the writer through `metadataTargetRoot` instead of `dest.absRoot`. Transfer returns a `note` when names couldn't be written (path unset); sync 400s with the reason.
- The writer itself unchanged — `<path>./Rom.gba</path>` still resolves (ES-DE resolves paths against the system ROM dir, not the gamelist location).

### UI
- Device page: when launcher = ES-DE, a "ES-DE gamelists folder" FolderPicker (rooted at the mount) + a `<details>` with step-by-step Android relocate instructions (user requested in-app guidance).
- Transfer page shows the `note` (warn) when names weren't written.
