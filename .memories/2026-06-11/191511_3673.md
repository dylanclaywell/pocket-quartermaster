## Multi-source pivot implemented (Steps 1–3 of 4)

Built the any-source→any-destination model. No master library anymore.

### Step 1 — catalog (computeLibrary)
Every configured+scanned source folds into games/variants (was: only the one library). `activeCaches` = caches of all configured sources. Each variant's `librarySources` accumulates every source holding it. Per-source presence matrix (`game.destinations`) now built for ALL configured sources, not just non-library ones. `effectiveLibraryKey`/role kept but vestigial (only feeds the `sources` summary). Verified: 390 games = union (174 on both sources, 216 on one); sample variant heldBy both 'My Passport' + 'RG Rotate SD'.

### Step 2 — /roms reframe
Replaced the master-library picker with a "Showing: [All sources | <source>]" filter. `/api/roms` list items gained `installedOn: string[]` (cache keys holding the game) to drive client-side filtering. Removed setLibrary/libraryKey/configuredSources. Verified: filter to RG Rotate SD → 174/390.

### Step 3 — any→any transfer
`buildTransferPlan(cfg, sourceCacheKey, destCacheKey, gameKeys)`: variant chosen from what the SOURCE holds (dest-preferred → game-default → first candidate). plan.post + execute.post take sourceCacheKey, reject source==dest (400). Transfer page shell now has Source + Destination dropdowns (source = scanned sources, dest = configured sources ≠ source); TransferRomsTab takes sourceCacheKey, watches both. Verified: RG Rotate SD → My Passport plan resolves, variant from source, already-installed flagged; src==dst → 400.

### Verification
typecheck clean after each step; dev server booted, endpoints + pages 200, no log errors. Servers killed each time.
