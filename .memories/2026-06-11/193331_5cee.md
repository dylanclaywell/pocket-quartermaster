## Step 4 done — art push with per-device resize (jimp)

Completes the bidirectional art pipeline. Server cache → ES-DE downloaded_media, optionally downscaled for small screens.

### Built
- Installed **jimp 1.6.1** (pure JS, Pi-safe). API: `Jimp.read(buf)`, `img.scaleToFit({w,h})`, `img.getBuffer("image/png")`.
- `server/utils/launcherArtPush.ts` — `pushLauncherArt(cfg, destCacheKey, gameKeys?, maxEdgeOverride?)`. For each game installed on the destination that has cached art, writes `<ES-DE>/downloaded_media/<system>/covers/<installed-rom-stem>.<ext>` (named after the ROM file so ES-DE matches it). If a max edge is set and art is larger, downscales longest edge via scaleToFit → re-encodes PNG; otherwise copies original bytes. Removes same-stem covers under other extensions to avoid stale dupes. Atomic tmp+rename.
- `server/api/roms/art/push.post.ts` — POST {destCacheKey, gameKeys?, maxEdgePx?}.
- Per-device `artMaxEdgePx` on DeviceIdentity + VirtualMount; SourceRoot carries it; both PATCH endpoints validate (positive int or null/0 to clear).
- Art tab (`TransferArtTab.vue`): shows count of installed-with-art games (+ how many lack art), a Max size (px) input (blank = device setting/full), and a Push button. Result note.
- Device page ES-DE section: "Push art max size (px)" input + Save.

### Verified
- typecheck clean.
- push endpoint: 400 no body; reason "Art push is only supported for ES-DE destinations" for non-es-de.
- Resize on a REAL cached cover: 512x512 (469KB) → 320x320 (211KB) PNG. scaleToFit caps longest edge, preserves aspect.
- Transfer page 200. Server killed.
- True device write (to a mounted ES-DE handheld) is the user's hardware test — not exercisable here.
