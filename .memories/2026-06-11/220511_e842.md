## muOS art (Step 2) — box-art push/harvest/reconcile, both launchers unified

Completes muOS support and generalizes the art pipeline across ES-DE + muOS.

### New shared layer
- `server/utils/launcherArt.ts` — `getArtLayout(kind, dest)` returns `{ boxDir(systemKey), harvestDirs }` per launcher (async; muOS reads the device's catalogue dir).
  - ES-DE: boxDir = `<downloaded_media>/<systemKey>/covers`; harvestDirs = each system's covers/miximages/screenshots.
  - muOS: boxDir = `<MUOS>/info/catalogue/<CatName>/box` where CatName comes from a static `MUOS_CATALOGUE_NAMES` map matched against the device's ACTUAL catalogue folders (exact, else normalized match); returns null if no such folder on device (skip, never guess-create). harvestDirs = every existing `catalogue/*/box`.
- Static map built from the user's real device listing (gb→"Nintendo Game Boy", snes→"Nintendo SNES - SFC", genesis→"Sega Mega Drive - Genesis", pce→"NEC PC Engine", ngp→"SNK Neo Geo Pocket - Color", etc.).

### Refactored to use the layout
- `pushLauncherArt` / `artPushPlan` (launcherArtPush.ts) and `harvestLauncherArt` (launcherArtHarvest.ts) now dispatch es-de|muos via getArtLayout; ES-DE behavior unchanged. Harvest now matches by ROM filename stem alone (gameKey is name-based, so folder/system-agnostic — works for muOS catalogue too). Push gained `skippedNoFolder`.

### Device page
muOS section expanded to "Names & art": Sync names + Import art + push-art max-size, mirroring ES-DE. Transfer Art tab already launcher-agnostic (uses /api/roms/art/plan + push) so muOS destinations work there with no change.

### Verified
- typecheck clean.
- ES-DE art-plan regression: still reconciles after refactor.
- muOS catalogue matcher simulated against the user's real folder names: every present system → correct folder; absent (nds, saturn) skip cleanly.
- True muOS device write = user hardware test.
