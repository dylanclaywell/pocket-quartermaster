## Vestigial cleanup + libretro art on ROM detail page

Two changes this session.

### 1. Removed dead single-master-library code
The multi-source pivot left these unused: deleted `server/api/roms/library.put.ts`; removed `RomLibraryRole` type, `ConfigFile.romLibrarySourceKey`, `effectiveLibraryKey`/`romConfiguredCacheKeys`, and the `role` field from SourceMeta/LibrarySource in romLibrary.ts; `buildSourceMeta` no longer takes a libraryKey. Removed `role` from the `/roms` page LibrarySummary interface. Existing config.json files may still contain `romLibrarySourceKey` — harmlessly ignored, no migration. Verified: /api/roms returns no `role` field, 405 games.

### 2. libretro box-art setting on the ROM game-detail page
Mirrors the activity detail page's thumbnail UI, on `/roms/[gameKey]`.
- `cores.ts`: new `SYSTEM_LIBRETRO_DB_NAMES` map (systemKey → libretro-thumbnails repo names), e.g. gba → ["Nintendo - Game Boy Advance"], pce → [PCE, SuperGrafx]. Added WSWAN_COLOR + NEOGEO_POCKET_BASE constants.
- `/api/roms/games/[gameKey]` now returns `libretroDbNames` (+ existing `hasThumbnail`).
- `/roms/[gameKey].vue`: new "Box art" section — debounced libretro search (`/api/thumbnails/search?systems=&q=`), result list with download, paste-URL fallback (`/api/thumbnails/download`), remove (`DELETE /api/thumbnails/<gameKey>`), cache-busting `?v=` on the img. Keys on gameKey = the shared thumbnail cache, so art set here shows in activity + library + pushes to launchers.

- GBA game detail → libretroDbNames ["Nintendo - Game Boy Advance"]; libretro search "mario" → 20 results, system ok.
- /api/roms regression clean (no role).
